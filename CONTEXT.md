# CONTEXT.md - 项目上下文

## 问题背景
在进行vibe coding时，Claude Code会在需要用户决策时等待响应。当用户把Claude Code窗口放在一旁时，可能因为专注于其他任务而错过了这些决策时机，导致效率降低。

## 解决方案
开发一个带GUI界面的屏幕活动监控程序，通过定期截取用户选择的屏幕区域并比较像素变化来检测画面是否静止。当检测到画面长时间没有变化（表示Claude Code可能在等待用户决策）时，发出可自定义的蜂鸣报警提醒。

## 关键设计决策
1. **GUI界面**: 使用tkinter构建，包含区域选择、预览、配置和监控控制
2. **区域选择**: 点击按钮后进入全屏选择模式，拖动鼠标选择监控区域
3. **区域预览**: 在主界面显示所选区域的缩略预览图
4. **屏幕捕获**: 使用mss库进行高性能屏幕截图
5. **活动检测**: 比较连续两帧画面的像素差异，如果差异小于阈值则认为画面静止
6. **报警机制**: 使用winsound.Beep生成不同频率/模式的蜂鸣
7. **多显示器支持**: 可选择监控哪个显示器

## 状态
v0.2.0版本发布，GUI版本稳定可用。
